# Roadmap

This roadmap is intentionally short and public-facing.

## Current public scope

- publish and stabilize the **full** line first
- keep the repository clean, understandable, and reproducible
- document the public direction of pyLoad-ft more clearly

## Ongoing priorities

- continue security hardening
- improve path and containment safety
- refine WebUI and API permission boundaries
- improve hardened container runtime behavior
- keep Docker builds reproducible and maintainable

## HTTPS / TLS direction

A major future area is cleaner native HTTPS/TLS behavior for containerized deployments.

This includes:

- more robust secure defaults
- clearer operational behavior
- better handling for hardened deployments

## Later scope

The separately developed **lite** line exists internally, but is not the first public publication target.
